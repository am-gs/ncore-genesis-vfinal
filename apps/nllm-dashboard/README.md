     1	# NLLM.ING Dashboard
     2	
     3	This is the Next.js dashboard for NLLM.ING, inspired by Manus.im's visual interface.
     4	
     5	## Features
     6	
     7	- Real-time agent progress monitoring
     8	- Interactive session management
     9	- Workspace file browser
    10	- Usage analytics and cost tracking
    11	- Dark/light theme support
    12	
    13	## Development
    14	
    15	To run the development server:
    16	
    17	```bash
    18	pnpm dev
    19	```
    20	
    21	Open [http://localhost:3002](http://localhost:3002) with your browser to see the result.
    22	
    23	## Architecture
    24	
    25	The dashboard follows a three-panel layout similar to Manus.im:
    26	
    27	1. **Sidebar** - Session history and navigation
    28	2. **Main Workspace** - Chat interface and task execution visualization
    29	3. **Detail Panel** - Usage statistics and file browser
    30	
    31	## API Integration
    32	
    33	The dashboard connects to the NCore orchestrator API at `/api` for task execution and real-time updates via WebSocket.
    34	
    35	## Implementation Summary
    36	
    37	The NLLM.ING Dashboard has been implemented as a Next.js application with the following key components:
    38	
    39	1. **Real-time WebSocket Integration**: The dashboard uses WebSockets to receive real-time updates from the orchestrator service, providing live feedback on agent progress and task execution.
    40	
    41	2. **Three-Panel Layout**: Following the Manus.im design pattern, the dashboard features a sidebar for navigation and session history, a main workspace for task execution visualization, and a detail panel for usage statistics and file browsing.
    42	
    43	3. **Core Components**:
    44	   - Task input and execution interface
    45	   - Session management with status indicators
    46	   - Agent progress cards showing real-time status
    47	   - Task planning visualization
    48	   - File browser with search functionality
    49	   - Usage analytics with data visualization
    50	   - Responsive design with dark/light theme support
    51	
    52	4. **API Integration**: The dashboard integrates with the NCore orchestrator API to submit tasks, retrieve session history, and access generated files.
    53	
    54	5. **NLLM.ING Branding**: The dashboard features the NLLM.ING branding with the tagline "Never-Refusing AI. Costs Pennies." and highlights the $0 routing tier.
    55	
    56	## How it maps to Manus.im's dashboard
    57	
    58	The NLLM.ING Dashboard implements many of the key features found in Manus.im's dashboard:
    59	
    60	1. **Interactive Session Monitoring**: Like Manus.im, the dashboard provides real-time updates on task execution progress, showing the status of individual agents as they work.
    61	
    62	2. **Workspace File Browser**: The dashboard includes a file browser component that allows users to view and manage files generated by tasks, similar to Manus.im's file management capabilities.
    63	
    64	3. **Usage Analytics**: The dashboard features usage analytics and cost tracking, providing insights into token consumption and expenses, similar to Manus.im's analytics dashboard.
    65	
    66	4. **Task History**: Users can view and manage their task sessions, with status indicators showing completed, active, and errored tasks.
    67	
    68	5. **Data Visualization**: The dashboard includes data visualization components for displaying usage statistics and performance metrics.
    69	
    70	6. **Responsive Design**: The dashboard is designed to be responsive and works well on different screen sizes, with a clean, minimal aesthetic that matches Manus.im's design language.
    71	
    72	## Screenshots Description
    73	
    74	1. **Main Dashboard**: The main dashboard view features the three-panel layout with the NLLM.ING logo and tagline in the sidebar, task input and session history in the left panel, agent progress visualization in the center panel, and usage statistics in the right panel.

    77

    79

    81	
    82	5. **Settings Page**: The settings page allows users to configure model preferences, API keys, and system settings, with a clean, organized interface for managing dashboard preferences.
